❯ winget 安装和使用npm install有什么区别，我能同时安装两个吗？                  

● 两种安装方式的主要区别：

  ┌───────────┬──────────────────────────┬─────────────────────────────────┐
  │  对比项   │          WinGet          │         npm install -g          │
  ├───────────┼──────────────────────────┼─────────────────────────────────┤
  │           │ 单个打包的               │ cli.js + Node.js 依赖（需要本机 │
  │ 文件形式  │ claude.exe（自带         │  Node.js）                      │
  │           │ Node.js）                │                                 │
  ├───────────┼──────────────────────────┼─────────────────────────────────┤
  │ 更新方式  │ winget upgrade           │ npm update -g                   │
  │           │                          │ @anthropic-ai/claude-code       │
  ├───────────┼──────────────────────────┼─────────────────────────────────┤
  │ 访问源码  │ 不能（已打包）           │ 可以直接看 cli.js               │
  ├───────────┼──────────────────────────┼─────────────────────────────────┤
  │ 依赖      │ 不需要                   │ 需要本机安装 Node.js            │
  │ Node.js   │                          │                                 │
  └───────────┴──────────────────────────┴─────────────────────────────────┘

  可以同时安装两个吗？

  技术上可以，但会造成冲突。两者都会注册 claude 命令，运行时取决于 PATH
  里谁排在前面，容易混乱，不建议同时安装。

  建议：
  - 只是正常使用 → 保持 WinGet 版本即可
  - 想看/修改源码 → 先卸载 WinGet 版本，再用 npm 安装

❯ 安装完nodejs之后，系统里到底安装了些什么？包括V8吗？                          

● 安装 Node.js 后，系统里包含：

  核心组件：
  - node.exe — Node.js 运行时（内嵌 V8 引擎）
  - npm.cmd / npx.cmd — 包管理器
  - corepack.cmd — 管理 yarn/pnpm 等工具

  是的，包含 V8 — V8 JavaScript 引擎直接编译进 node.exe 内部，不是单独的 .dll
  文件，无法独立调用。

  其他内置库（也编译进 node.exe）：
  - libuv — 异步 I/O 事件循环
  - llhttp — HTTP 解析器
  - zlib — 压缩
  - OpenSSL — 加密/TLS

  默认安装路径（Windows）：
  C:\Program Files\nodejs\
  ├── node.exe
  ├── npm
  ├── npx
  └── node_modules\    ← npm/npx 的核心模块

  全局 npm 包路径：
  C:\Users\你的用户名\AppData\Roaming\npm\

❯ 那么使用fnm之类的包管理工具的作用是什么？既然都已经安装好了nodejs了           

● fnm（或 nvm）的核心作用是管理多个 Node.js 版本。

  主要使用场景：

  - 项目 A 需要 Node.js 18，项目 B 需要 Node.js 20 → 随时切换
  - 团队统一版本（.nvmrc / .node-version 文件锁定版本）
  - 测试代码在不同 Node.js 版本下的兼容性
  - 升级/降级 Node.js 不用去官网下载安装包

  如果你只有一个项目、不需要切换版本，直接装 Node.js 官方安装包就够了，不需要
  fnm/nvm。

  fnm 相比 nvm 的优势是用 Rust 写的，速度更快，Windows 支持更好。

❯ 我已经安装好了nodejs,还需要再使用fnm吗？                                      

● 不需要。

  你已经安装好了 Node.js，如果没有多版本切换的需求，直接用就好，不用再装 fnm。
  
❯ 如果继续安装了fnm,那么对于我已经安装好的nodejs版本，是在fnm的管理控制码？ 
                                                                            
● 不在。                                                                                                                                                  fnm 安装后只管理它自己下载的 Node.js                                        版本，不会接管你之前通过官方安装包装的 Node.js。                          
                                                                            
  两者会并存，具体用哪个取决于 PATH
  的顺序。这也是为什么不建议混用——容易造成版本混乱。        

  如果你以后想用 fnm 统一管理，正确做法是：
  1. 卸载官方安装的 Node.js
  2. 用 fnm 重新安装同版本
