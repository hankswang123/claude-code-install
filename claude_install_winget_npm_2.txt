❯ npm 版本和claude code版本的区别                                               

● Claude Code 有两种主要安装方式，它们的版本是同步的，功能完全一致：

  1. npm 安装 (你当前使用的)

  npm install -g @anthropic-ai/claude-code

  特点：
  - 依赖 Node.js 环境
  - 跟随 Node 版本管理器（fnm/nvm）
  - 切换 Node 版本后可能需要重装
  - 适合已有 Node.js 开发环境的用户

  2. 独立安装包

  macOS:
  brew install claude-code

  Windows:
  winget install Anthropic.ClaudeCode

  特点：
  - 自带运行时，不依赖系统 Node.js
  - 更新独立，不受 Node 版本影响
  - 安装更简单

  版本对比

  ┌──────────┬────────────────────────────────────┬─────────────────────────┐
  │   方面   │              npm 版本              │       独立安装版        │
  ├──────────┼────────────────────────────────────┼─────────────────────────┤
  │ 版本号   │ 同步                               │ 同步                    │
  ├──────────┼────────────────────────────────────┼─────────────────────────┤
  │ 功能     │ 完全相同                           │ 完全相同                │
  ├──────────┼────────────────────────────────────┼─────────────────────────┤
  │ 更新方式 │ npm update -g                      │ brew upgrade / winget   │
  │          │ @anthropic-ai/claude-code          │ upgrade                 │
  ├──────────┼────────────────────────────────────┼─────────────────────────┤
  │ 依赖     │ 需要 Node.js ≥18                   │ 无额外依赖              │
  └──────────┴────────────────────────────────────┴─────────────────────────┘

  你现在的 2.1.83 版本就是最新的，无论哪种安装方式都是同一个版本。选择哪种主要看
  个人偏好和环境需求。

❯ 我的另外一台电脑上显示v2.1.81, 不过claude code是使用winget 安装的             

● 这就解释了！

  npm 和 winget 的版本发布不完全同步。

  ┌──────────┬──────────┬─────────────────────────────────────────┐
  │ 安装渠道 │ 当前版本 │                更新命令                 │
  ├──────────┼──────────┼─────────────────────────────────────────┤
  │ npm      │ 2.1.83   │ npm update -g @anthropic-ai/claude-code │
  ├──────────┼──────────┼─────────────────────────────────────────┤
  │ winget   │ 2.1.81   │ winget upgrade Anthropic.ClaudeCode     │
  └──────────┴──────────┴─────────────────────────────────────────┘

  为什么会有版本差异？

  1. 发布流程不同 - npm 发布由 Anthropic 直接控制，几乎即时生效；winget 需要经过
   Microsoft 的审核流程，通常会延迟几天
  2. 打包时间差 - 独立安装包需要额外的构建和签名步骤

  如何更新 winget 版本？

  在另一台电脑上运行：
  winget upgrade Anthropic.ClaudeCode

  如果还是 2.1.81，说明 winget 源还没同步到最新版，等几天就会有了。

  功能上 2.1.81 和 2.1.83 差异很小，通常是 bug 修复，正常使用不受影响。

❯ 但是使用winget安装之后，我只看到一个claude.exe文件，没有其它文件              

● 对，这就是关键区别。

  winget 安装 vs npm 安装

  ┌──────────┬─────────────────────────────────────────────┬────────────────┐
  │   方面   │                   winget                    │      npm       │
  ├──────────┼─────────────────────────────────────────────┼────────────────┤
  │ 文件结构 │ 单个打包的 claude.exe                       │ 完整的 JS      │
  │          │                                             │ 源码目录       │
  ├──────────┼─────────────────────────────────────────────┼────────────────┤
  │ 可修改性 │ ❌ 几乎不可能                               │ ✅ 直接编辑 JS │
  │          │                                             │  文件          │
  ├──────────┼─────────────────────────────────────────────┼────────────────┤
  │ 原理     │ 使用 pkg/nexe 等工具把 Node.js +            │ 原始 Node.js   │
  │          │ 代码打包成单个可执行文件                    │ 模块           │
  └──────────┴─────────────────────────────────────────────┴────────────────┘

  winget 的 claude.exe 结构

  claude.exe = Node.js 运行时 + cli.js + 所有依赖 (打包压缩)

  这是一个二进制打包文件，无法直接编辑里面的 JS 代码。

  结论

  如果你需要修改 Claude Code（比如运行你的 patch 脚本），必须使用 npm 安装。

  在另一台电脑上操作：

  # 1. 卸载 winget 版本
  winget uninstall Anthropic.ClaudeCode

  # 2. 确保有 Node.js >= 18
  node --version

  # 3. npm 全局安装
  npm install -g @anthropic-ai/claude-code

  # 4. 应用你的补丁
  python patch_claude.py

  这样两台电脑就都可以使用你的 patch 脚本了。
